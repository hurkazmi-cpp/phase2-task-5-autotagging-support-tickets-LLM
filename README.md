# Task 5: Auto Tagging Support Tickets Using LLM

## 📌 Project Objective
In high-volume customer support operations, manually reading, categorizing, and routing incoming free-text support tickets is time-consuming and highly prone to human error. The objective of this task is to develop an automated support ticket tagging system driven by an instruction-tuned Large Language Model (LLM). 

This project explores:
* Custom structural prompt engineering frameworks for multi-class classification.
* Comparative evaluation of **Zero-Shot Learning** vs. **Few-Shot Learning** methodologies.
* Surfacing multi-class predictions by ranking and returning the **Top 3** most probable operational tags per ticket.

---

## ⚙️ Methodology & Approach

The pipeline implemented inside the Jupyter Notebook follow a strict production-ready lifecycle:

1. **Dataset Loading & Preprocessing**: 
   A structured dataset is built to simulate realistic customer service inquiries encompassing distinct categories such as `Billing Issue`, `Technical Bug`, `Account Access`, `Feature Request`, `Password Reset`, and `Hardware Failure`.

2. **Zero-Shot Inference Framework**:
   The LLM is prompted out-of-the-box with strict systemic instructions defining the bounding context, the allowed list of candidate tags, and a forced constraints formatting blueprint. No historical classifications are leaked to the context window.

3. **Few-Shot Inference Framework**:
   The context window is enriched by injecting highly descriptive, static few-shot demonstration exemplars. These pairs guide the LLM's syntactic style and align its decision boundaries for edge-case ticket tracking.

4. **Parsing & Ranked Selection Engine**:
   A deterministic token parser processes the raw string output generated by the LLM pipeline, extracts the returned elements, matches them against permitted labels, and selects the Top 3 choices.

---

## 📈 Key Results & Observations

To evaluate the operational success of both methodologies, the system measures the **Primary Tag Retrieval Accuracy**, validating whether the correct highest-priority ground-truth tag was successfully recovered within the model's predicted Top 3 list.

| Methodology | Primary Tag Retrieval Accuracy (Top 3) |
| :--- | :--- |
| **Zero-Shot Learning** | `83.3%` |
| **Few-Shot Learning** | `100.0%` |

### Major Insights:
* **Structural Compliance**: Zero-shot prompting provides an exceptional semantic baseline but occasionally struggles with output formatting constraints.
* **Contextual Alignment**: Few-shot learning successfully mitigates classification errors by providing structural anchor points, ensuring complete deterministic alignment and an optimized accuracy profile.
* **Triage Efficiency**: Generating a ranked list of the top 3 candidate tags provides an excellent balance for automated routing, routing tickets immediately to specialized support branches while providing fallbacks.
